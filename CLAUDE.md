## 功能清單

### 卡片學習系統
- 星星進度條：完成一句點亮一顆 ⭐，完成全集點亮 🌟
- 語塊 chunk-pill：動詞=綠、名詞=藍、形容詞=紫，點語塊 → TTS 西語朗讀
- 點整句 → TTS 念整句
- 翻譯欄位：智慧比對（中文/英文，關鍵詞重疊 ≥55% 算對）
- 看答案：中 / EN 兩 tab，附文法說明

### YouGlish 發音
- `cognates.js` 的 `SENTENCE_YG_KW` 定義每句的精準搜尋短語
- 按鈕 `yg-card-btn` → `openYGPanel(keyword)` → `window.open` 開新分頁
- URL 格式：`https://youglish.com/pronounce/{encoded}/spanish`

### 英西同源槓桿
- 每句卡片下方有 `<details>` 摺疊寶箱
- 資料在 `cognates.js` 的 `SENTENCE_COGNATES`，key 格式 `e{ep}_s{idx}`
- `buildCogDetails(data)` 產生 HTML

### 造句區
- 顯示句型骨架（`getMakePattern`）
- E1–E6 每句皆有心智圖換詞地圖（`expand` 資料，`renderExpandMap`）
- 核對句型：比對關鍵動詞，成功 → TTS 念出使用者句子

### 彈藥庫（ammo.js）
- `AMMO_DATA` 陣列，每張彈藥卡：核心彈藥 → 武器改裝 → 實戰射擊
- `SENTENCE_AMMO_MAP2`：globalIdx → ammo_id(s) 對應
- 完成句子 → `unlockAmmo(globalIdx)` → 自動解鎖

### LocalStorage
- key：`peppa_es_v4`
- 儲存：`{ ammoUnlocked, ammoStars }`

- [ ] **📚 ND 家長情緒調節策略研究（2026-07-08 蒐集，未落地成功能，先當內容素材庫）**：VERA 要求聚焦「ND 家長帶 ND 寶，大人自己的調整模式」而非泛泛的育兒技巧，之後可能用來充實「聊療吾心語」或「心田深耕」的內容素材：

  - **前額葉調節機制（VERA 特別要求的角度）**：情緒失調的神經機轉是前額葉（尤其背外側前額葉 DLPFC）對杏仁核的「由上而下」抑制訊號不足；ADHD 族群這條前額葉—杏仁核連結效率較低，這是為什麼「先講道理」常常沒用——要先讓前額葉重新上線。
  - **有神經科學實證的具體技巧**：
    - **情緒命名（affect labeling）**：「我現在注意到焦慮」「這裡有生氣的感覺」——單純講出情緒標籤就能量測到杏仁核活化下降，是啟動前額葉調節迴路最容易上手的方式（比認知重評更適合情緒當下爆表時使用）
    - **認知重評（cognitive reappraisal）**：重新詮釋情境的意義，會活化外側前額葉並抑制杏仁核反應，效果甚至能持續到一週後
    - **正念/專注呼吸**：8 週正念減壓（MBSR）訓練後，杏仁核對情緒刺激的反應實測下降，且前額葉—杏仁核的連結強度增加（結構性改變，不只是當下感覺變好）
    - **STOP 技巧**：Stop（停）→ Take a breath（呼吸）→ Observe（覺察身體/念頭/情緒）→ Proceed（帶著覺察繼續）——幾秒鐘就能重新接上前額葉調節
    - 自我疼惜（self-compassion）：承認自己在做很難的事、掙扎不等於失敗，研究證實能提升韌性、降低照顧者過勞
  - **雙重神經多樣性（ND 家長×ND 孩子）特有洞察**：雙重同理問題（Double Empathy Problem）——連結變好是因為「雙方都學著翻譯」，不是只逼孩子單方面配合神經典型社交規則；ND 家長優勢是同理心深、能用過動式超專注策略性投入陪伴。

## 給 Claude Code 的工作守則

1. **手機優先**：所有 UI 改動先想手機版，max-width: 480px
2. **不破 CSS 變數**：顏色一律用 `var(--konpeki)` 等變數，不要 hardcode（除非 debug 用）
3. **語塊教學法**：禁止單字死背，所有教學內容以語塊/片語/完整句為單位。**這條包含一個具體設計原則（2026-07-12 記錄）：一整段補語子句包成單一 O 語塊是刻意設計，不是遺漏**——例如 ammo.js 的「una persona muy buena encontrando cosas.」這種長補語，不會再往下拆成內部自己的 S/V/O，因為 O 語塊本來就可以代表「受詞/補語，也用於整句包框」（見 episodes.js 開頭圖例）。之後如果又被問到「這句是不是少了 SVO」，直接回答這是既定設計，不用重新查證。

5. **LocalStorage key 是 peppa_es_v4**：不要改 key 名稱，會讓用戶資料消失

## 系統性重要備註（給接手的人）

- **CDN雙層快取**：CSS/JS靠?v=xxx版本號破快取；HTML本體也有自己的CDN快取層，不受版本號保護，改完通常要等30-60秒才會同步，遇到「明明推了但沒變」先等久一點再判斷。**⚠️ 鐵則（2026-07-12 教訓）：每次改完 style.css 或任何 .js 檔，一定要同步把 index.html 裡對應的 `?v=xxx` 版本號往上跳一版，兩者必須同一個 commit 一起推**——曾經連續好幾個 commit 只改了 js/css 內容、忘記跳版號，導致 VERA 那幾天「怎麼推都看不到任何變化」，因為瀏覽器/CDN 一直在吃舊版本號對應的快取檔案，版本號沒變就沒有理由重新抓檔案。
- **語塊配色系統**：S=虛線框、V=實色底、O=實線框，色票變數在style.css開頭:root（--v-navy、--o-deepblue等），之前被覆蓋過一次，改動前建議先確認現況。
- **隱含主詞一律用西語**：不要改回中文括號。
- **GitHub Actions workflow**：.github/workflows/deploy.yml目前只跑GitHub Pages，Netlify job已經想拿掉但token權限不足沒推成功，如果之後要處理需要有workflow權限的token。
